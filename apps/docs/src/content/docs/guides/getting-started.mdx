## Why JSSON? ‚ú®

JSSON makes writing JSON feel natural. No more wrestling with quotes, commas, and brackets. Write configuration files the way you think about them ‚Äî clean, readable, and human-friendly.

**What makes JSSON special:**
- **No quotes needed** for simple values
- **Template arrays** turn spreadsheet-like data into structured JSON
- **Map transformations** for powerful data manipulation
- **Ranges** generate sequences automatically
- **Includes** split large configs into manageable files
- **Expressions** compute values on the fly

## Installation

<Steps>

1. **Download the JSSON binary**

   Head to the [GitHub releases page](https://github.com/carlosedujs/jsson/releases) and download the latest version for your platform.

2. **Make it executable** (Linux/macOS)

   ```bash
   chmod +x jsson
   ```

3. **Move to your PATH** (optional but recommended)

   ```bash
   # Linux/macOS
   sudo mv jsson /usr/local/bin/

   # Windows: Add the directory to your PATH environment variable
   ```

4. **Verify installation**

   ```bash
   jsson --version
   ```

</Steps>

> **Building from source?** If you have Go installed, you can build JSSON yourself:
> ```bash
> git clone https://github.com/carlosedujs/jsson
> cd jsson
> go build -o jsson cmd/jsson/main.go
> ```

## Your First JSSON File

Let's create a simple configuration file. Create a new file called `config.jsson`:

<Code lang="jsson" code={`// My first JSSON file
app {
  name = "My Awesome App"
  version = "1.0.0"
  debug = true
}

server {
  host = localhost
  port = 8080
}`} />

Notice how clean this is? No quotes around keys, no commas, no fuss!

## Transpile to JSON

Now let's convert this to JSON:

```bash
jsson -i config.jsson -o config.json
```

This creates `config.json` with perfect, valid JSON:

<Code lang="json" code={`{
  "app": {
    "name": "My Awesome App",
    "version": "1.0.0",
    "debug": true
  },
  "server": {
    "host": "localhost",
    "port": 8080
  }
}`} />

**That's it!** You just wrote and transpiled your first JSSON file. üéâ

## Common First-Time Mistakes

### ‚ùå Forgetting Quotes for Multi-Word Values

```jsson
// Wrong
title = Software Engineer

// Right
title = "Software Engineer"
```

### ‚ùå Using Commas Between Fields

```jsson
// Wrong
name = Jo√£o,
age = 25,

// Right
name = Jo√£o
age = 25
```

JSSON doesn't need commas! Just put each field on a new line.

### ‚ùå Mixing Syntax Styles

```jsson
// Wrong - mixing object styles
user {
  name = Jo√£o
  config = { theme: "dark" }  // Don't use JSON syntax inside JSSON
}

// Right
user {
  name = Jo√£o
  config {
    theme = dark
  }
}
```

## Quick Reference

| Feature | JSSON | JSON |
|---------|-------|------|
| **Keys** | `name = value` | `"name": value` |
| **Strings** | `Jo√£o` or `"Jo√£o"` | `"Jo√£o"` |
| **Objects** | `obj { ... }` | `"obj": { ... }` |
| **Arrays** | `[ a, b, c ]` | `["a", "b", "c"]` |
| **Comments** | `// comment` | ‚ùå Not supported |
