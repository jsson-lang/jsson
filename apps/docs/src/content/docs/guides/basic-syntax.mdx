## Core Concepts

JSSON has just a few simple rules. Master these, and you'll be writing clean configs in no time.

### Assignments

The foundation of JSSON: key-value pairs.

<Code lang="jsson" code={`name = JoÃ£o
age = 25
active = true`} />

**Rules:**
- No quotes needed around keys
- Use `=` for assignment
- One assignment per line (no commas!)

### When to Use Quotes

This is the #1 question! Here's the simple rule:

<Tabs>
  <TabItem label="No Quotes Needed">
    <Code lang="jsson" code={`// Single words
name = JoÃ£o
city = Tokyo
status = active

// Numbers
age = 25
price = 19.99

// Booleans
admin = true
disabled = false`} />
  </TabItem>

  <TabItem label="Quotes Required">
    <Code lang="jsson" code={`// Values with spaces
title = "Software Engineer"
message = "Hello, World!"

// Values with special characters
email = "user@example.com"
url = "https://example.com"

// Empty strings
empty = ""`} />
  </TabItem>
</Tabs>

> **Pro Tip:** When in doubt, use quotes! They always work. But skipping them makes your JSSON cleaner.

## Comments

Document your configs with comments â€” they're ignored during transpilation.

<Code lang="jsson" code={`// This is a single-line comment

// Application settings
app {
  name = "My App"  // Inline comments work too!
  version = "1.0.0"
  
  // Server configuration
  server {
    port = 8080  // Default port
    host = localhost
  }
}`} />

> **Pro Tip:** Use comments to explain *why* a setting exists, not just *what* it is.

## Building from Simple to Complex

Let's build a real-world example step by step.

### Step 1: Basic Info

### Step 2: Add Objects

<Code lang="jsson" code={`name = "My Application"

### Step 3: Add Arrays

### Step 4: Add Nesting

See how natural it feels? No JSON ceremony, just clean configuration!
