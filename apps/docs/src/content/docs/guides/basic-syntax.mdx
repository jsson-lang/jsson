## Core Concepts

JSSON has just a few simple rules. Master these, and you'll be writing clean configs in no time.

### Assignments

The foundation of JSSON: key-value pairs.

<Code
  lang="javascript"
  code={`name := JoÃ£o
age := 25
active := true`}
/>

**Rules:**

- No quotes needed around keys
- Use `=` for assignment and `:=` for variable declarations
- One assignment per line (no commas!)

### When to Use Quotes

This is the #1 question! Here's the simple rule:

<Tabs>
  <TabItem label="No Quotes Needed">
    <Code lang="javascript" code={`// Single words
name := JoÃ£o
city := Tokyo
status := active

// Numbers
age := 25
price := 19.99

// Booleans
admin := true
disabled := false`} />

  </TabItem>

  <TabItem label="Quotes Required">
    <Code lang="javascript" code={`// Values with spaces
title := "Software Engineer"
message := "Hello, World!"

// Values with special characters
email := "user@example.com"
url := "https://example.com"

// Empty strings
empty := ""`} />

  </TabItem>
</Tabs>

> **Pro Tip:** When in doubt, use quotes! They always work. But skipping them makes your JSSON cleaner.

## Comments

Document your configs with comments â€” they're ignored during transpilation.

<Code lang="javascript" code={`// This is a single-line comment

// Application settings
app {
name = "My App" // Inline comments work too!
version = "1.0.0"

// Server configuration
server {
port = 8080 // Default port
host = localhost
}
}`} />

> **Pro Tip:** Use comments to explain _why_ a setting exists, not just _what_ it is.

## Variables

Declare variables with `:=` to store values for reuse. Variables are **not** included in the final JSON output.

<Tabs>
  <TabItem label="JSSON">
    <Code lang="javascript" code={`// Declare variables
api_key := "sk-1234567890"
base_url := "https://api.example.com"
timeout := 5000

// Use them in configuration
production {
apiKey = api_key
endpoint = base_url + "/v1"
timeout = timeout
}

development {
apiKey = api_key
endpoint = base_url + "/dev"
timeout = timeout
}`} />

  </TabItem>

  <TabItem label="JSON Output">
    <Code lang="json" code={`{
  "production": {
    "apiKey": "sk-1234567890",
    "endpoint": "https://api.example.com/v1",
    "timeout": 5000
  },
  "development": {
    "apiKey": "sk-1234567890",
    "endpoint": "https://api.example.com/dev",
    "timeout": 5000
  }
}`} />
  </TabItem>
</Tabs>

### Local Variables

Variables declared inside objects are **local** to that scope:

<Code lang="javascript" code={`global := 100

obj {
local := 50
value = local // Uses local (50)
outer = global // Uses global (100)
}

// 'local' is not accessible here`} />

> **Use Case:** Variables are perfect for configuration constants, API keys, and any value you use multiple times.
